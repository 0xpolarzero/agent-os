# Prompt: Bundle Common WASM Runtime Artifacts

You are working in an `agent-os` worktree. Act as an orchestrator and use subagents heavily for source inspection, packaging validation, and verification.

## Goal

Make the fork directly consumable as a dependency source tree by ensuring the common registry packages ship their required `wasm/` runtime payloads.

## Context

- The JS wrappers for common software packages resolve binaries from `../wasm`.
- Upstream docs say those directories are normally generated by `make copy-wasm` and are gitignored.
- That is fine for upstream source development, but it breaks vendored dependency consumption because the exported packages exist without their runtime payloads.

## Required Work

1. Confirm which packages are required by `@rivet-dev/agent-os-common`.
2. Ensure those packages have tracked `wasm/` directories in the fork.
3. Update ignore rules narrowly so the intended runtime payloads remain versioned.
4. Do not change command registration code unless validation proves it is wrong.
5. Verify that a source consumer can boot Agent OS with `common` and execute shell commands.

## Constraints

- Prefer the smallest branch that makes the fork runnable.
- Do not introduce a fallback that silently reaches back to published npm packages at runtime.
- Keep documentation concise but explicit about why this branch carries generated artifacts.

## Verification

- Confirm the `wasm/` directories exist for the common packages.
- Run the minimal runtime verification needed to show `sh` or `echo` is available from a source checkout.
- Report any residual limitations separately from the completed fix.
